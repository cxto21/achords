Instalación

Instala Achords y configura tu primera organización.

Prerrequisitos

  • Node.js 18+
  • Git
  • GitHub CLI (gh) autenticado

Instalar via npm

npm install -g achords

Verificar

achords version

Quick Start

Bootstrap una organización y onboardea tu primer repo en 5 minutos.

1. Crear tu org

achords obase --org my-company

Esto crea:

~/achords/my-company/
├── .achords/      # Reglas de agentes
├── .skills/       # Skills compartidos
├── .internal/     # Onboarding de devs
└── .github/       # Perfil de org

2. Onboardear un repo

cd ~/achords/my-company
achords obase --repo my-app

3. Verificar

ls -la my-app/
# Debería mostrar .engram/
cat my-app/AGENTS.md
# Debería mostrar instrucciones de agente

Conceptos

Estructura de Organización

Una organización es el contenedor principal. Contiene:

~/achords/{org-name}/
├── .achords/           # Reglas de agentes
│   ├── AGENTS.md       # Reglas para todos
│   ├── .engram/        # Memoria compartida
│   ├── conventions.json
│   └── policies.json
├── .skills/            # Skills compartidos
│   ├── AGENTS.md
│   ├── version.json
│   └── skills/
├── .internal/          # Onboarding
│   ├── README.md
│   ├── AGENTS.md
│   └── scripts/
├── .github/            # Perfil org
│   └── profile/
│       └── README.md
└── repos/              # Tus proyectos
    ├── app-1/
    └── app-2/

Integración de Repos

Los repos importan reglas de la org via git submodules:

my-app/
├── .achords → ../.achords/ (submodule)
├── .skills  → ../.skills/  (submodule)
├── .engram/                (memoria repo)
├── AGENTS.md               (instrucciones)
├── src/
└── ...

Actualizar reglas

cd ~/achords/my-company/my-app
git submodule update --remote .achords .skills

Aislamiento de Memoria

Org memory (compartida)
└── .achords/.engram/
    └── Todos los repos la ven

Repo memory (aislada)
└── .engram/
    └── Solo este repo la ve
Nivel Ubicación Contenido
Org .achords/.engram/ Convenciones, arquitectura, patrones cross-repo
Repo .engram/ Bugs, features, testing discoveries

Flujo del Agente

Sesión start
    ↓
mem_context (recuperar contexto)
    ↓
Leer .achords/AGENTS.md (obligatorio)
    ↓
Leer .skills/AGENTS.md (obligatorio)
    ↓
Trabajar...
    ↓
mem_save (después de decisiones/fixes)
    ↓
mem_session_summary (antes de cerrar)

Skills

Skills siguen el Agent Skills spec.

SKILL.md

---
name: my-skill
description: Qué hace este skill
triggers:
  - cuándo usar este skill
scope:
  - "**/*.ts"
  - "**/*.js"
---

# My Skill

## Instructions

1. Paso uno
2. Paso dos

## Resources

- [Link](url) — Descripción

Versionado

Cada versión es un directorio:

.skills/
└── skills/
    └── testing/
        ├── manifest.json
        └── versions/
            ├── v1.0.0/
            │   └── SKILL.md
            └── v1.1.0/
                └── SKILL.md

manifest.json

{
  "name": "testing",
  "description": "Testing patterns",
  "versions": {
    "v1.0.0": {
      "created": "2025-01-15",
      "platforms": ["linux", "windows"]
    }
  }
}

Crear Skills

# 1. Crear directorio
mkdir -p .skills/skills/my-skill/versions/v1.0.0

# 2. Crear SKILL.md
cat > .skills/skills/my-skill/versions/v1.0.0/SKILL.md << 'EOF'
---
name: my-skill
description: Qué hace
---
# My Skill
## Instructions
1. Paso uno
EOF

# 3. Crear manifest
cat > .skills/skills/my-skill/manifest.json << 'EOF'
{
  "name": "my-skill",
  "description": "Qué hace",
  "versions": {
    "v1.0.0": {
      "created": "2025-03-01",
      "platforms": ["linux"]
    }
  }
}
EOF

Memoria (Engram)

Memoria persistente via Engram.

APIs principales

API Propósito
mem_save Guardar memoria
mem_search Buscar memorias
mem_context Obtener contexto reciente
mem_session_summary Resumen de sesión

Protocolo

Cuándo guardar

  • Decisión de arquitectura
  • Fix de bug
  • Descubrimiento no obvio
  • Cambio de configuración
  • Patrón establecido

Formato

{
  "title": "Título corto y buscable",
  "type": "decision|bugfix|discovery|pattern|config",
  "content": {
    "What": "Qué se hizo",
    "Why": "Qué lo motivó",
    "Where": "Archivos afectados",
    "Learned": "Gotchas y edge cases"
  }
}

Topic Keys

Agrupan memorias relacionadas:

architecture/auth-model
bugfix/n-plus-one-query
config/engram-setup

Mismo topic_key + mismo proyecto = upsert (actualiza existente).

achords obase

Inicializa una organización o configura un repo.

Flag Descripción Default
--org Nombre de la org Requerido
--repo Repo a configurar Ninguno
--dir Directorio custom ~/achords/{org}
--skills URL del repo de skills Ninguno
--update-headers Actualizar AGENTS.md en todos los repos false
--push Push cambios a GitHub false

Ejemplos

# Inicializar org
achords obase --org my-company

# Configurar repo existente
cd ~/achords/my-company
achords obase --repo my-app

# Actualizar todos los repos
achords obase --org my-company --update-headers
